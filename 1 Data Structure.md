# Data Structure and Algorithms

**Data Structures and Algorithms (DSA) in C** is the study of how to **organize data (Data Structures)** and **process data efficiently (Algorithms)** using the **C Programming Language**.

Understanding DSA deeply will help you write **optimized and efficient programs**.

---

# 1. What is DSA?

### Data Structure

A **Data Structure** is a way of **storing and organizing data in memory** so that it can be accessed and modified efficiently.

Example:

* Array
* Linked List
* Stack
* Queue
* Tree
* Graph
* Hash Table
* Heap

### Algorithm

An **Algorithm** is a **step-by-step procedure to solve a problem**.

Example:

* Searching algorithms
* Sorting algorithms
* Graph traversal

So,

**DSA = Data Structure + Algorithm**

---

# 2. Types of Data Structures

Data structures are mainly divided into two categories.

## 1. Linear Data Structures

Elements are arranged **sequentially (one after another)**.

Examples:

* Array
* Linked List
* Stack
* Queue

Structure:

```
A → B → C → D → E
```

---

## 2. Non-Linear Data Structures

Elements are arranged **hierarchically or interconnected**.

Examples:

* Tree
* Graph
* Heap
* Hash Table

Structure:

```
       A
     /   \
    B     C
   / \   / \
  D   E F   G
```

# 4. Searching Algorithms

---

## Linear Search

Search element sequentially.

Time Complexity:

O(n)



## Binary Search

Works only on **sorted arrays**.

Time Complexity:

O(log n)

---

# 5. Sorting Algorithms

Sorting arranges elements in **ascending or descending order**.

Common sorting algorithms:

1. Bubble Sort
2. Selection Sort
3. Insertion Sort
4. Merge Sort
5. Quick Sort
6. Heap Sort

Time Complexity

| Algorithm      | Complexity |
| -------------- | ---------- |
| Bubble Sort    | O(n²)      |
| Selection Sort | O(n²)      |
| Insertion Sort | O(n²)      |
| Merge Sort     | O(n log n) |
| Quick Sort     | O(n log n) |
| Heap Sort      | O(n log n) |

---

# 6. Time Complexity

Time complexity measures **how algorithm time grows with input size**.

Common complexities:

| Complexity | Meaning     |
| ---------- | ----------- |
| O(1)       | Constant    |
| O(log n)   | Logarithmic |
| O(n)       | Linear      |
| O(n log n) | Efficient   |
| O(n²)      | Slow        |
| O(2ⁿ)      | Very slow   |

Example

```
Binary Search → O(log n)
Linear Search → O(n)
Bubble Sort → O(n²)
```

---

# 7. Space Complexity

Space complexity measures **memory usage of algorithm**.

Example

```
Extra array used → more space
In-place sorting → less space
```
